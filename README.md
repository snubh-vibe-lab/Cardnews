# Cardnews

SNUBH Vibe Lab 카드뉴스 저장소입니다. 중견 임상의가 실제 진료·연구·교육·행정 워크플로우에 AI 도구를 적용할 수 있도록, 짧고 실행 가능한 카드뉴스 주제를 축적합니다.

## 현재 반영된 카드뉴스 묶음

- Webdev: 6장
- Agent: 4장
- MCP: 4장
- Paper: 4장
- Project: 4장
- Schedule: 4장
- Troubleshoot: 4장
- Airgap/Context 보강 카드: 각 1장

## Vibe Lab 맞춤 카드뉴스 토픽 9개

1. **나만의 임상 AI 비서 만들기**
   - 외래, 판독, 연구, 행정에서 반복되는 일을 작은 에이전트로 나누는 방법
   - 예시: 메일 정리, 일정 요약, 학회 업무 추적, 교육 현황 체크

2. **논문 읽기에서 연구 아이디어까지**
   - PubMed/arXiv 검색 → 핵심 요약 → 내 데이터셋에 맞는 연구질문으로 바꾸는 흐름
   - 예시: 영상의학 AI 논문을 내 병원 연구 주제로 변환하기

3. **의료 데이터를 안전하게 다루는 로컬/폐쇄망 AI**
   - 환자정보, IRB, 비식별화, 온프레미스 LLM, air-gapped workflow 기본 개념
   - 예시: 민감정보를 외부 서비스에 올리지 않는 분석 구조

4. **MCP와 병원 업무 자동화**
   - 여러 도구·DB·문서·캘린더를 에이전트가 연결해 쓰는 방식
   - 예시: Gmail, Calendar, Drive, GitHub, 병원 포털을 하나의 업무 흐름으로 묶기

5. **임상의가 직접 만드는 연구용 웹앱**
   - 코딩을 깊게 몰라도 데이터 입력폼, 대시보드, 리뷰 도구를 만드는 방법
   - 예시: 증례 리뷰 앱, 연구 대상자 관리 앱, 카드뉴스 생성 앱

6. **프롬프트보다 중요한 컨텍스트 설계**
   - 좋은 답변을 얻기 위해 배경, 목표, 제약, 예시, 출력 형식을 구조화하는 법
   - 예시: “요약해줘”가 아니라 “학회 보고용 1페이지로 정리해줘”라고 지시하기

7. **AI 에이전트로 일정·메일·학회 업무 운영하기**
   - Zia/Zuli/Zulu/Zapil/Zebe처럼 역할별 에이전트를 나누어 운영하는 패턴
   - 예시: 학회 결재, KCR 메일, 워크숍 참석 회신, 병원 교육 관리

8. **의료진을 위한 디버깅 사고법**
   - AI가 틀렸을 때 원인 파악, 로그 확인, 재현, 검증하는 방법
   - 예시: 토큰 만료, 브라우저 세션 끊김, 자동화 실패를 안전하게 복구하기

9. **작은 자동화에서 팀 운영으로 확장하기**
   - 개인 도구를 팀 표준 워크플로우, GitHub repo, 템플릿, 체크리스트로 확장하는 법
   - 예시: Vibe Lab 공통 카드뉴스, 온보딩, 월례 워크숍 자료 관리

## 운영 메모

- 각 토픽은 4~6장 카드뉴스 단위로 제작합니다.
- 대상 독자는 “AI에 관심은 있지만 개발자가 아닌 중견 임상의”로 둡니다.
- 모든 카드뉴스는 실제 병원/연구/학회 업무 예시를 포함합니다.
- 민감정보, 환자정보, 계정정보는 카드뉴스 예시에도 포함하지 않습니다.
