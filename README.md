## 다음 카드뉴스 후보: 바이브 코딩 & Agentic AI 실전 팁 9개

> 기존 월례모임 발표 주제형 안은 폐기하고, 짧은 카드뉴스로 바로 소비할 수 있는 팁 중심으로 재작성했습니다.

1. **처음부터 만들지 말고 “고칠 파일”부터 찍어주기**
   - “이 기능 만들어줘”보다 “이 파일의 이 함수만 고쳐줘”가 결과가 좋습니다.
   - 팁: 파일명, 함수명, 기대 출력 1개를 같이 줍니다.

2. **요구사항은 3줄로 쪼개기**
   - 목적 / 제약 / 확인방법을 분리하면 AI가 덜 헤맵니다.
   - 예시: “목적: 버튼 추가, 제약: 기존 CSS 유지, 확인: npm test 통과.”

3. **한 번에 완성보다 작은 PR처럼 시키기**
   - 큰 기능은 “1단계: 뼈대, 2단계: 테스트, 3단계: 화면 정리”로 나눕니다.
   - 팁: 각 단계 끝에 diff와 실행결과를 요구합니다.

4. **AI에게 먼저 코드 읽기만 시키기**
   - 바로 수정시키기 전에 “관련 파일 찾고 구조 요약만 해줘”라고 시킵니다.
   - 효과: 엉뚱한 파일을 고치는 실수를 줄입니다.

5. **테스트 명령을 모르면 AI에게 찾게 하기**
   - “테스트 돌려줘”보다 “package/README/CI에서 테스트 명령 찾아 실행해줘”가 안전합니다.
   - 팁: 실패 로그를 그대로 붙여 다시 고치게 합니다.

6. **Agent에게 권한 경계를 먼저 말하기**
   - “파일 수정 가능, 배포 금지, 삭제 전 확인”처럼 허용/금지 행동을 선명히 둡니다.
   - Agentic AI는 자율성이 있으므로 금지선을 먼저 정하는 것이 중요합니다.

7. **로그·스크린샷·에러문구를 같이 주기**
   - “안 돼”보다 실제 에러 10줄, 화면 캡처, 재현 순서를 주면 해결률이 올라갑니다.
   - 팁: 에러 전체가 길면 마지막 50줄부터 줍니다.

8. **결과 보고는 ‘무엇을 바꿨고 어떻게 확인했는지’로 받기**
   - 좋은 보고 형식: 변경 파일 / 실행 명령 / 결과 / 남은 위험.
   - “잘 됐습니다”만 말하면 검증이 빠진 것입니다.

9. **자주 쓰는 지시는 템플릿으로 저장하기**
   - 반복 업무는 매번 프롬프트를 새로 쓰지 말고 템플릿화합니다.
   - 예시: “버그 수정 템플릿”, “카드뉴스 생성 템플릿”, “회의자료 작성 템플릿”.

## 운영 메모

- 각 토픽은 1장 또는 2장짜리 짧은 카드뉴스로 제작합니다.
- 대상 독자는 “AI에 관심은 있지만 개발자가 아닌 중견 임상의”로 둡니다.
- 월례모임 발표 주제라기보다, 메신저에서 바로 읽히는 실전 팁 톤을 유지합니다.
- 기존 이미지 묶음과 중복되지 않도록 구체적 사용 팁·작업 습관 중심으로 확장합니다.
- 민감정보, 환자정보, 계정정보는 카드뉴스 예시에도 포함하지 않습니다.
